# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in the Bangla Calendar project, please email [security contact] instead of using the issue tracker.

**Please do not disclose the vulnerability publicly until we've had a chance to fix it.**

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Security Updates

We take security vulnerabilities seriously and will:

1. Acknowledge receipt of your report within 48 hours
2. Provide an estimated timeline for a fix
3. Release a fix and credit you (if desired)
4. Publish a security advisory

## Dependencies

We regularly update dependencies and run security audits:

- Weekly automated dependency checks via GitHub Actions
- Regular manual security reviews
- Automated security scanning via Dependabot

## Best Practices

When contributing code:

- Don't commit sensitive data (API keys, credentials, etc.)
- Follow secure coding practices
- Review dependencies before adding new packages
- Keep TypeScript strict mode enabled
